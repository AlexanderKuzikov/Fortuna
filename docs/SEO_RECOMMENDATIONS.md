# Рекомендации по улучшению SEO

> **Дата:** 09.07.2026
> **Проект:** Fortuna — сайт компании по декорированию искусственными растениями

---

## Критические проблемы

### 1. Sitemap: несоответствие URL

**Проблема:** В `sitemap.xml` URL указаны без trailing slash (`/catalog`), но `.htaccess` настроен на добавление слеша и перенаправление. Это создает дубли страниц и путаницу для поисковых систем.

**Решение:**
- Исправить все URL в `sitemap.xml` на версию со слешем:
  - `/catalog` → `/catalog/`
  - `/works` → `/works/`
  - `/about` → `/about/`
  - `/delivery` → `/delivery/`
  - `/contacts` → `/contacts/`

**Приоритет:** 🔴 Высокий

---

### 2. Favicon и иконки

**Проблема:** Отсутствует `<link rel="icon">` в `<head>`. Сайт не имеет иконки в закладках браузера.

**Решение:**
```html
<link rel="icon" type="image/svg+xml" href="/favicon.svg">
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">
<link rel="manifest" href="/site.webmanifest">
<meta name="msapplication-TileColor" content="#f5f3ee">
```

**Приоритет:** 🔴 Высокий

---

### 3. Open Graph изображения

**Проблема:** Не все страницы имеют `og:image`. При репостах в соцсетях отсутствует превью.

**Решение:**
- Создать уникальные OG images для каждой страницы (1200×630px)
- Добавить `og:image:width` и `og:image:height`
- Убедиться, что изображения доступны по абсолютному URL

**Приоритет:** 🔴 Высокий

---

### 4. Структурированные данные для продуктов

**Проблема:** На всех страницах используется только `LocalBusiness` schema. Нет разметки для конкретных товаров/услуг.

**Решение:**
- Добавить `Product` schema для каждой категории товаров
- `Service` schema для услуг (озеленение, фитостены)
- `ItemList` schema для каталога
- Пример для товаров:
```json
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Букет искусственных пионов",
  "category": "Искусственные цветы",
  "material": ["шелк", "латекс"],
  "offers": {
    "@type": "Offer",
    "priceCurrency": "RUB",
    "availability": "https://schema.org/InStock"
  }
}
```

**Приоритет:** 🔴 Высокий

---

### 5. Canonical URL: неконсистентность

**Проблема:** В некоторых страницах canonical указан без ведущего слеша, в некоторых — относительный путь.

**Решение:**
- Использовать полные абсолютные URL:
  - `<link rel="canonical" href="https://fortuna-perm.com/catalog/"/>`
- Проверить все страницы на консистентность

**Приоритет:** 🔴 Высокий

## Проблемы среднего приоритета

### 6. Локальное SEO (Пермь)

**Проблема:** Сайт ориентирован на Пермь, но geo-теги отсутствуют. Это снижает позиции в локальной выдаче.

**Решение:**
```html
<meta name="geo.region" content="RU-PERM">
<meta name="geo.placename" content="Пермь">
<meta name="geo.position" content="58.0105;56.2502">
<meta name="ICBM" content="58.0105, 56.2502">
<meta name="yandex-verification" content="...">
<meta name="google-site-verification" content="...">
```

**Дополнительно:**
- Зарегистрироваться в Яндекс.Справочник
- Google Business Profile
- 2ГИС справочник
- Собирать и публиковать отзывы на площадках

### 7. Внутренняя перелинковка

**Проблема:** Слабая связь между страницами. Категории каталога не ссылаются на проекты, где использовались эти растения, и наоборот.

**Решение:**
- На страницах категорий каталога добавить блок "Примеры работ с этими растениями"
- На страницах проектов — блок "Использованные материалы"
- На главной — ссылки на популярные категории и проекты
- Улучшить навигацию между связанными страницами

### 8. Meta descriptions

**Проблема:** Некоторые meta descriptions обрезаны или неоптимальной длины.

**Решение:**
- Привести все описания к 150-160 символам
- Включить ключевые слова + CTA
- Уникальное описание для каждой страницы
- Использовать формат: "Что? Где? (Пермь). Почему стоит выбрать? CTA."

### 9. BreadcrumbList schema

**Проблема:** Хлебные крошки есть не на всех страницах, а Schema.org разметка для них отсутствует.

**Решение:**
```
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {"@type": "ListItem", "position": 1, "name": "Главная", "item": "/"},
    {"@type": "ListItem", "position": 2, "name": "Каталог", "item": "/catalog/"},
    {"@type": "ListItem", "position": 3, "name": "Цветы", "item": "/catalog/iskusstvennye-tsvety/"}
  ]
}
```

### 10. Robots.txt

**Проблема:** Файл слишком минималистичный (2 строки). Нет ссылки на sitemap, нет правил для разных типов роботов.

**Решение:**
```
User-agent: *
Allow: /
Disallow: /404.html
Crawl-delay: 10

Sitemap: https://fortuna-perm.com/sitemap.xml
```

### 11. H1 заголовки

**Проблема:** Проверить, что на каждой странице ровно один H1 и он содержит ключевые слова.

**Решение:**
- Уникальный H1 для каждой страницы
- Включение ключевых слов (Пермь, искусственные цветы и т.д.)
- Оптимальная длина 40-70 символов

## Проблемы низкого приоритета

### 12. Контент-маркетинг (блог)

- Добавить блог/новости на отдельный поддомен или раздел
- Темы: "Как ухаживать за искусственными растениями", "Тренды озеленения", "Выбор растений для кафе"
- Регулярность: 2-4 статьи в месяц
- Увеличит: органический трафик, время на сайте, E-E-A-T

### 13. FAQPage structured data

- Создать FAQ на страницах категорий
- Schema.org/FAQPage разметка
- Rich snippets с вопросами/ответами в поиске

### 14. Видео-контент

- Добавить видео обзоры товаров и проектов
- Схема VideoObject для видео
- YouTube плейлисты

### 15. Image SEO

- Улучшить alt-тексты для всех изображений
- Описательные имена файлов (не cat-flowers-DN_DkEQ_.jpg, а iskusstvennye-piony-buket.jpg)
- WebP формат с fallback на JPG
- Sitemap для изображений

### 16. Core Web Vitals

- Оптимизировать LCP (крупный текст/изображение)
- Оптимизировать CLS (явные размеры изображений)
- INP: минимизировать задержки ввода (JS)
