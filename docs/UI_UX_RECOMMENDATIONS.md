# Рекомендации по улучшению UI/UX

> **Дата:** 09.07.2026
> **Проект:** Fortuna — сайт компании по декорированию искусственными растениями

---

## Критические проблемы

### 1. Мобильное меню

**Проблема:** Навигация скрыта на мобильных устройствах (`hidden lg:flex`). Бургер-меню отсутствует. Пользователи на телефонах не могут перемещаться по сайту.

**Решение:**
- Реализовать hamburger menu для экранов < 1024px
- Анимация открытия/закрытия
- Затемнение фона при открытом меню
- Крупные touch-friendly ссылки (min 44px)

**Приоритет:** 🔴 Высокий

---

### 2. Футер

**Проблема:** `<footer>` отсутствует на всех страницах. Пользователи не могут найти контакты, соцсети, политику конфиденциальности внизу страницы.

**Решение:**
```
┌─────────────────────────────────────┐
│  Логотип  │  Контакты  │  Соцсети   │
│           │  Тел.: ... │  VK        │
│           │  Email: .. │  WhatsApp  │
│           │  Адрес     │  Telegram  │
├─────────────────────────────────────┤
│  Навигация: Каталог, Работы, О нас  │
├─────────────────────────────────────┤
│  © 2026 Fortuna  |  Политика конф.  │
└─────────────────────────────────────┘
```

**Приоритет:** 🔴 Высокий

---

### 3. Изображения товаров

**Проблема:** На страницах категорий каталога только текстовое описание. Нет фотографий конкретных товаров, нет цен, нет размеров.

**Решение:**
- Добавить сетку товаров с фото
- Карточка товара: фото, название, цена, размер, кнопка "Заказать"
- Lightbox для увеличения изображений
- Фото с разных ракурсов

**Приоритет:** 🔴 Высокий

---

### 4. Призывы к действию (CTA)

**Проблема:** На большинстве страниц нет кнопок, побуждающих к действию. Пользователь не знает, что делать дальше.

**Решение:**
- На каждой странице добавить CTA:
  - "Заказать консультацию"
  - "Позвонить"
  - "Написать в WhatsApp"
  - "Смотреть каталог"
- Фиксированная кнопка WhatsApp в углу экрана
- Floating phone button на мобильных

**Приоритет:** 🔴 Высокий

---

### 5. Отсутствие хлебных крошек на некоторых страницах

**Проблема:** На некоторых страницах (контакты, доставка, о нас) нет breadcrumbs. Пользователь не понимает, где находится.

**Решение:**
- Добавить breadcrumbs на все страницы
- Schema.org BreadcrumbList разметка

**Приоритет:** 🔴 Высокий

## Проблемы среднего приоритета

### 6. Доступность (Accessibility)

**Проблема:** Сайт не проходит базовые проверки доступности.

**Решение:**
- Добавить `aria-label` для всех интерактивных элементов
- `role="navigation"` для nav
- `role="banner"` для header
- Улучшить контрастность текста
- Focus visible styles для клавиатурной навигации
- Skip to content ссылка

### 7. Производительность изображений

**Проблема:** Все изображения в JPG формате, нет адаптивных размеров.

**Решение:**
- Конвертировать в WebP (с потерями ~80% качества)
- Добавить `srcset` с разными разрешениями
- `loading="lazy"` для всех изображений кроме hero
- `width` и `height` атрибуты для предотвращения CLS

### 8. Микровзаимодействия

**Проблема:** Отсутствие анимаций и反馈 реакций на действия пользователя.

**Решение:**
- Hover эффекты на карточках проектов
- Плавный scroll (scroll-behavior: smooth)
- Анимации появления элементов при скролле
- Состояния загрузки (skeleton screens)
- Микроанимации для кнопок

### 9. Социальное доказательство

**Проблема:** Нет отзывов клиентов, рейтингов, примеров работ с описанием результатов.

**Решение:**
- Блок отзывов на главной
- Карусель с отзывами
- Счетчик выполненных проектов
- Логотипы клиентов (кафе, рестораны)

### 10. Форма обратной связи

**Проблема:** На странице контактов только телефон и email. Нет формы для быстрой связи.

**Решение:**
- Форма: имя, телефон, email, сообщение
- Валидация на клиенте (без JS — HTML5 атрибуты)
- Интеграция с Telegram/email
